Google Test - Part 1

Dissecting our first test program using Google Test (gtest)

#include "gtest/gtest.h"

TEST(LearnGoogleTest, Test1) {
}

int main(int argc, char* argv[]) {
  ::testing::InitGoogleTest(&argc, argv);
  return RUN_ALL_TESTS();
}
  • The test program starts by including the header file “gtest/gtest.h”.
  • The ::testing::InitGoogleTest(&argc, argrv) call parses the commandline gtest flags to the test program and removes all unrecognised flags. It should be called before the call to macro RUN_ALL_TESTS()
  • RUN_ALL_TESTS() macro call runs all the test in the given program.
  • Each test is defined in the TEST(test_case_name, test_name) macro block. For e.g.
TEST(Factorial, zero) {
   EXPECT_EQ(factorial(0), 1);
  }

EXPECT_EQ(val1, val2) is a assertion macro which tests for val1 == val2

Compiling and Running a test program

#include "gtest/gtest.h"

TEST(LearnGoogleTest, Test1) {
  EXPECT_EQ(1, 1);
}

int main(int argc,  char* argv[]) {
  ::testing::InitGoogleTest(&argc, argv);
  return RUN_ALL_TESTS();
}

By virtue of using the gtest framework, the test program’s behavior can be controlled by command line flags. To see the commandline flags available

following command line flags to control its behavior:

Test Selection:
  --gtest_list_tests
      List the names of all tests instead of running them. The name of

The commandline flags can be also be controlled programmatically as follows:

int main(int argc,  char* argv[]) {
  ::testing::FLAGS_gtest_color = "no"; // turns off the coloring of the test output
  ::testing::FLAGS_gtest_print_time = 0; // this will turn off printing the elapsed test time
  ::testing::InitGoogleTest(&argc, argv);
  return RUN_ALL_TESTS();
}
